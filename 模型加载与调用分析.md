## 一、整体架构

项目是**离线英译中 Markdown 编辑器**，核心翻译能力来自两个独立模型：

| 模型 | 用途 | 技术栈 | 数据量 |
|------|------|--------|--------|
| **Argos Translate（NMT）** | 段落级神经网络翻译 | CTranslate2 + Argos Translate | ~67MB |
| **ECDICT（词典）** | 单词级悬停翻译 | SQLite 只读查询 | ~50MB（77 万词条） |

两者互补：神经网络负责完整句子理解，词典负责即时应答。

#### `_unwrap_translation()` 的作用

**文件：** `src/translate/engine.py:6-13`

```python
def _unwrap_translation(t):
    from argostranslate.translate import CachedTranslation, PackageTranslation
    while isinstance(t, CachedTranslation):
        t = t.underlying
    if not isinstance(t, PackageTranslation):
        raise TypeError(...)
    return t
```

递归剥除缓存包装层。`CachedTranslation` 是 Argos 内部对翻译结果做缓存的对象，其 `underlying` 属性指向真正的 `PackageTranslation`。必须剥除才能访问底层 `translator`、`sentencizer`、`tokenizer` 等组件。

## 三、翻译调用方式

### 3.1 单句翻译

**文件：** `src/translate/engine.py:51-54` — `translate()`

```python
def translate(self, text: str) -> str:
    if not text or not text.strip():
        return ""
    return self._t.translate(text)
```

走 Argos 高层封装，内部流程：
```
self._t.translate(text)
  └── PackageTranslation.translate()
        ├── sentencizer.split_sentences()
        ├── tokenizer.encode()
        ├── translator.translate()       ← 单次推理
        └── tokenizer.decode()
```

适用于 Worker 逐条翻译场景。

### 3.2 批量翻译（核心优化）

**文件：** `src/translate/engine.py:56-124` — `translate_many()`

这是整个项目**最重要的性能优化**。手动实现完整 pipeline，将所有段落的句子合并到一次 `translate_batch` 调用。

```
输入: ["Hello world.", "How are you?", "I'm fine."]
                        │
     for each text:
       ├── sentencizer.split_sentences(text)
       │     "Hello world." → ["Hello world."]
       │     "How are you?\nI'm fine." → ["How are you?", "I'm fine."]
       └── tokenizer.encode(sentence) → ["▁Hello", "▁world", "."]

     boundaries = [1, 1, 2]       ← 每段对应几个句子
     all_tokenized = [
       ["▁Hello", "▁world", "."],
       ["▁How", "▁are", "▁you", "?"],
       ["▁I", "'m", "▁fine", "."],
     ]
                        │
     translator.translate_batch(
       all_tokenized,
       target_prefix=[[target_prefix_text]] × N,  # 可选
       replace_unknowns=True,
       max_batch_size=0,       # 0 = 不限制
       batch_type="tokens",    # 按 token 数动态分 batch
       beam_size=1,            # 贪心解码
       num_hypotheses=1,
       length_penalty=0.2,
       return_scores=True,
     )
                        │
     按 boundaries 重构段落:
       for count in boundaries:
         for _ in range(count):
           收集 hypotheses[0] tokens
         tokenizer.decode(tokens) → 译文文本
         去掉 target_prefix 前缀
         去掉首字符空格
```

#### 参数详解

| 参数 | 值 | 说明 |
|------|----|------|
| `beam_size` | 1 | 贪心搜索，最快但可能不是最优翻译 |
| `num_hypotheses` | 1 | 只保留最佳候选 |
| `length_penalty` | 0.2 | 轻微偏向较长句子（经验值） |
| `replace_unknowns` | True | 将 `<unk>` 替换为原文词 |
| `max_batch_size` | 0 | 由 `batch_type` 动态决定 |
| `batch_type` | "tokens" | 按 token 数量自动分 batch |

## 四、ECDICT 词典加载与调用

### 4.1 模型加载

**文件：** `src/translate/dict_cache.py:15-26`

```python
class DictCache:
    def __init__(self, db_path: Path):
        self._db_path = db_path
        self._conn: sqlite3.Connection | None = None

    def _connect(self) -> sqlite3.Connection:
        if self._conn is None:
            if not self._db_path.exists():
                raise FileNotFoundError(f"Dictionary not found: {self._db_path}")
            self._conn = sqlite3.connect(
                f"file:{self._db_path}?mode=ro", uri=True, check_same_thread=False
            )
        return self._conn
```

- **只读模式** (`mode=ro`)：防止误写入
- **延迟连接**：首次查询才打开数据库
- **`check_same_thread=False`**：允许从 QThread 访问（虽然目前悬停翻译在主线程）

### 4.2 查询逻辑

**文件：** `src/translate/dict_cache.py:28-38`

```python
def lookup(self, word: str) -> WordEntry | None:
    if not word or not word.isascii():
        return None
    cur = self._connect().execute(
        "SELECT word, translation, phonetic FROM words WHERE word = ? COLLATE NOCASE LIMIT 1",
        (word,),
    )
    row = cur.fetchone()
    if row is None:
        return None
    return WordEntry(row[0], row[1] or "", row[2] or "")
```

- **`COLLATE NOCASE`**：大小写不敏感（`"hello"` = `"HELLO"`）
- **`isascii()` 过滤**：非 ASCII 字符直接跳过
- **性能**：测试要求在 100 次查询中单次平均 < 30ms

### 5.2 LRU 缓存

**文件：** `src/translate/worker.py:25-41`

```python
class _LRU:
    def __init__(self, maxsize: int = 512):
        self._d: OrderedDict[str, str] = OrderedDict()

    def get(self, key: str) -> str | None:
        if key in self._d:
            self._d.move_to_end(key)    # 标记为最近使用
            return self._d[key]
        return None

    def set(self, key: str, value: str) -> None:
        self._d[key] = value
        self._d.move_to_end(key)
        while len(self._d) > self._maxsize:
            self._d.popitem(last=False)  # 淘汰最久未用
```

- Key 是段落的 MD5 哈希（节省内存）
- 最多缓存 512 条
- 每次访问将条目移到最后，淘汰表头条目

**文件：** `src/translate/worker.py:58-59,75-83`

## 八、关键设计决策

| 决策 | 说明 |
|------|------|
| **两层翻译互补** | NMT 做段落级理解，词典做单词级快速查询，互不抢占 |
| **批量翻译优化** | `translate_many()` 将所有段落的句子合并到一次 CTranslate2 `translate_batch`，避免逐段串行 |
| **ctranslate2 延迟加载** | 构造函数不加载模型，首次翻译前才初始化，减少启动时间 |
| **QThread + 队列** | 翻译在独立线程执行，不阻塞 UI；队列 + drain 32 条批处理 |
| **LRU + Inflight 去重** | 防止用户连续编辑时重复翻译同一段落 |
| **提前 import ctranslate2** | 在 Qt 修改 DLL 搜索路径之前加载 ctranslate2，避免 Windows 上 `c10.dll` 加载失败 |
| **资源缺失优雅降级** | 模型缺失 → 禁用翻译视图；词典缺失 → 禁用悬停，状态栏提示 |
| **信号防抖** | Worker 发出的 translated 信号不立即更新 UI，而是 150ms 防抖后批量刷新 |
| **段落保护** | `split_paragraphs()` 识别 fenced code block（```），不拆分代码块内部 |

## 九、潜在优化点

1. **`_load_translator()` 代码重复** — `translate()` 和 `translate_many()` 中有完全相同的 9 行 ctranslate2.Translator 构造代码，应提取为共享方法
2. **Worker 未使用 `translate_many()`** — Worker 逐条调用 `engine.translate(item)`，没有利用跨段落合并 batch 的能力
3. **无模型下载引导** — 模型缺失时直接禁用翻译视图，未提示用户运行 `fetch_argos_model.py`
4. **短语翻译缺失** — 悬停翻译只查单个单词，`"look up"`、`"break down"` 等短语无法翻译
5. **无翻译进度指示** — 段落翻译进行中时只显示 `…`，没有 loading 动画
